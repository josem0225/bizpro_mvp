B. Listas de Verificación para el Desarrollador (Lo que debes entregar)
El documento lista 5 "Blocking Issues" (Problemas Bloqueantes). Para que el cliente te apruebe y pague, debes cumplir con estos Checklists de Implementación:

1. Gestión de Archivos (Portal de Descargas)
Debes crear un componente en Softr conectado a Airtable que cumpla con esto:

[ ] Conectar a la tabla "Resources" de Airtable.

[ ] Implementar filtro: Mostrar solo archivos del paso actual (Step = current) Y que estén activos (IsActive = TRUE) Y coincidan con el idioma del cliente.

[ ] Botón "Preview" (Vista previa): Solo para PDFs.

[ ] Botón "Download" (Descargar): Para todos los tipos de archivo (PDF, DOCX, XLSX).

[ ] Agrupar archivos por categorías (Templates, Guides, Checklists, etc.).

[ ] Diseño responsivo en móviles (tarjetas apiladas verticalmente).

2. Lógica de Acceso al Paso 0 (Gratis vs Pago)
El diseño actual era confuso. Debes implementar dos rutas distintas:

[ ] Crear ruta pública /step-0 (accesible sin login).

[ ] Crear ruta autenticada /dashboard/step-0 (para usuarios logueados).

[ ] Landing Page: Botón "View Step 0 Free" lleva a la ruta pública; Botón "Get Started Free" lleva al Wizard de pago.

[ ] El Paso 0 debe permitir descargar 2 PDFs sin pedir email ni login.

[ ] Sidebar del Dashboard: Debe ocultar los pasos bloqueados (no mostrarlos en gris, sino ocultarlos).

3. Automatización del Desbloqueo de Pasos
Debes programar la lógica de cuándo se abre el siguiente paso:

[ ] Regla del 80%: El botón "Unlock Step X+1" solo aparece cuando el usuario completa el 80% del checklist del paso actual.

[ ] Al hacer clic en "Unlock", el estado cambia de bloqueado a desbloqueado inmediatamente.

[ ] Enviar email automático al desbloquear (vía Zapier/Airtable).

[ ] Implementar recordatorio de inactividad a los 7 días (email automático).

[ ] Panel Admin: Permitir desbloqueo manual por parte de soporte.

4. Características de Administración de Precios
Faltaban herramientas para el administrador. Debes crear 3 paneles nuevos:

[ ] Editor de Copia del Paywall: Editor de texto enriquecido que soporte variables como ${price} y ${step_name}.

[ ] Gestor de Enlaces de Stripe: Panel para crear/editar/archivar los 7 enlaces de pago. Debe validar que el link empiece por https://buy.stripe.com/.

[ ] Dashboard de Referencia de Precios: Panel de solo lectura que muestre precios actuales, métricas de ingresos (últimos 30 días) y ahorro por paquete.

[ ] Log de Auditoría: Crear tabla inmutable PriceHistory en Airtable que registre cada cambio de precio (quién, cuándo, valor anterior, valor nuevo).

[ ] Regla del 20%: Si se cambia un precio más del 20%, el sistema debe pedir confirmación.


Resumen de esfuerzo: El cliente estima que esto tomará ~15 días y recomienda trabajar del 12 al 28 de febrero.
